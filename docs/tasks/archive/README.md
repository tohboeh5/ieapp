## Archive Naming Convention

- `{milestone-name}.md` - Main milestone document with checkpoints and summary
- Milestones are named descriptively (e.g., `mvp-milestone`, `full-config-milestone`)

## Milestone Structure

Each archived milestone document includes:

1. **Summary** - What was delivered
2. **Checkpoints** - Completed tasks organized by phase
3. **Technical Decisions** - Key architecture choices made
4. **Lessons Learned** - Insights for future development
5. **Reference** - Links to roadmap and next milestones
